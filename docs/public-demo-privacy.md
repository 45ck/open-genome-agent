# Public Demo Privacy And Consent Boundaries

Public-demo work still needs privacy discipline even when the source profile is openly shared.

## Allowed

- use a public profile that clearly exposes participant data for open reuse
- cite the profile URL and selected public data row in local docs
- generate local run artifacts that point back to the public source

## Not allowed by default

- commit the downloaded genome file into this repository
- rehost a participant genome file from this repository
- blur self-reported traits into deterministic genomic findings
- present a public-demo report as diagnosis, screening, or treatment advice

## Required report language

- state that the genome file came from a public Harvard PGP profile
- state which file type and build were used
- keep limitations near any high-impact claim
- keep trait or PRS-style sections clearly separated from direct variant findings

## Why this boundary exists

Open sharing terms make the sample suitable for reproducible demos, but they do not remove the need for:

- local-first handling
- provenance
- non-diagnostic framing
- explicit uncertainty

Public availability is not a license to lower the evidence bar.
